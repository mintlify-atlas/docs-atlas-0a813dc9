The bot includes a multi-strategy engine that can run multiple technical analysis strategies simultaneously. Each strategy evaluates incoming candle data independently and emits a directional signal — LONG, SHORT, or NONE.

## How the engine works

The bot evaluates all active strategies per candle and takes the highest-confidence signal. Conflicting signals are filtered to avoid entering a position where strategies disagree on direction.

This means you can enable several strategies at once without them interfering. The engine aggregates their outputs and only acts when there is sufficient agreement or a clearly dominant signal.

## Enabling and disabling strategies

Each strategy has its own block inside the `strategies` object in `config.json`. Set `"enabled": true` to activate it or `"enabled": false` to disable it.

```json config.json
{
  "strategies": {
    "ema_crossover": { "enabled": true, "fast": 9, "slow": 21 },
    "rsi_divergence": { "enabled": true, "period": 14, "overbought": 70, "oversold": 30 },
    "bollinger_breakout": { "enabled": false, "period": 20, "std_dev": 2.0 },
    "macd_momentum": { "enabled": true, "fast": 12, "slow": 26, "signal": 9 }
  }
}
```

You can also configure strategy parameters directly from the terminal menu by selecting **Strategy Configuration** (option 6).

## Strategy defaults

| Strategy | Default state | Key parameters |
|---|---|---|
| EMA Crossover | Enabled | fast=9, slow=21 |
| RSI Divergence | Enabled | period=14, overbought=70, oversold=30 |
| Bollinger Band Breakout | **Disabled** | period=20, std_dev=2.0 |
| MACD Momentum | Enabled | fast=12, slow=26, signal=9 |

<Note>
  Bollinger Band Breakout is disabled by default. It performs best in high-volatility markets and should be tuned before enabling in production.
</Note>
