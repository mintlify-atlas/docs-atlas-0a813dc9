<Warning>
  Always test with **testnet** before trading real funds. Set `"testnet": true` in `config.json` to use Binance or Bybit testnet endpoints. Futures trading carries substantial risk of loss, including full account liquidation.
</Warning>

## Prerequisites

- Python 3.10 or higher
- pip (latest recommended)
- Exchange API keys with futures trading permissions
- A signal source: Telegram bot token, TradingView webhook URL, or a custom API endpoint

## Setup

<Steps>
  <Step title="Clone the repo and install dependencies">
    Clone the repository and install the required Python packages.

    <CodeGroup>
      ```bash Linux / macOS
      git clone https://github.com/Whit1985/Binance-Futures-Signal-Bot.git
      cd Binance-Futures-Signal-Bot
      pip install -r requirements.txt
      ```

      ```batch Windows
      git clone https://github.com/Whit1985/Binance-Futures-Signal-Bot.git
      cd Binance-Futures-Signal-Bot
      pip install -r requirements.txt
      ```
    </CodeGroup>

    Alternatively, skip the manual install and let the bot handle it. Run the launcher for your platform — it will automatically create a virtual environment and bootstrap dependencies:

    <CodeGroup>
      ```batch Windows
      run.bat
      ```

      ```bash Linux / macOS
      chmod +x run.sh
      ./run.sh
      ```
    </CodeGroup>

    Once the menu loads, choose option **1 — Install Dependencies**.
  </Step>

  <Step title="Configure API keys in config.json">
    Create or edit `config.json` in the project root. At minimum, fill in your exchange, API key, and API secret. The full set of available options is shown below.

    ```json config.json
    {
      "exchange": "binance",
      "api_key": "YOUR_API_KEY",
      "api_secret": "YOUR_API_SECRET",
      "leverage": 10,
      "max_position_pct": 5.0,
      "trailing_stop_pct": 1.5,
      "signal_source": "telegram",
      "telegram_bot_token": "BOT_TOKEN",
      "telegram_channel_id": "-100XXXXXXXXXX",
      "tradingview_webhook_port": 8080,
      "strategies": {
        "ema_crossover": {"enabled": true, "fast": 9, "slow": 21},
        "rsi_divergence": {"enabled": true, "period": 14, "overbought": 70, "oversold": 30},
        "bollinger_breakout": {"enabled": false, "period": 20, "std_dev": 2.0},
        "macd_momentum": {"enabled": true, "fast": 12, "slow": 26, "signal": 9}
      },
      "pairs": ["BTCUSDT", "ETHUSDT", "SOLUSDT"],
      "risk_management": {
        "max_open_positions": 5,
        "max_daily_loss_pct": 3.0,
        "stop_loss_pct": 2.0,
        "take_profit_pct": 4.0
      },
      "testnet": true
    }
    ```

    You can also use the interactive **Settings** menu (option **2**) to review all available configuration fields and their expected formats.

    <Tip>
      Never commit `config.json` to version control. It is included in `.gitignore` by default. On Linux and macOS, restrict file permissions with `chmod 600 config.json` to prevent other users from reading your API keys.
    </Tip>
  </Step>

  <Step title="Connect a signal source">
    Launch the bot and choose option **4 — Connect Signal Source** to configure where trade signals come from.

    ```bash
    python main.py
    ```

    Three sources are supported:

    - **Telegram** — Provide a bot token (from [@BotFather](https://t.me/BotFather)) and the channel ID (starts with `-100`). Add the bot to the signal channel as an admin.
    - **TradingView** — The bot starts a local webhook server on `tradingview_webhook_port` (default `8080`). Point your TradingView alert webhook URL at `http://<your-ip>:8080`.
    - **Custom API** — Connect a proprietary signal endpoint through the custom API adapter.

    See [Signal Sources](/signals/overview) for detailed setup instructions for each provider.
  </Step>

  <Step title="Start Auto Trading">
    Select option **5 — Start Auto Trading** from the main menu. The bot will begin monitoring your signal source and executing trades on the configured exchange.

    ```bash
    python main.py
    ```
